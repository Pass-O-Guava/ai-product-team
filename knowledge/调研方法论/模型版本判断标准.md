# 模型版本判断标准与方法论

> 来源：2026-04-07 内部研究整理
> 背景：WY 指出团队缺乏版本判断能力，调研结果中出现了大量过时信息

---

## 一、核心判断原则

### 原则1：版本只进不退，替代即为过时

- **GPT-4o** 发布 → GPT-4o 2024-05 旧版本自动标记为「历史版本」
- **Claude 3.7 Sonnet** 发布 → Claude 3.5 Sonnet 自动降为「次新版」
- **没有理由**因为新版好就删旧版，但**必须有理由**才保留旧版

### 原则2：「当前标杆」入库，「历史标杆」可选

| 模型状态 | 入库态度 | 标注要求 |
|---------|---------|---------|
| 当前最新版本 | ✅ 优先入库 | 无特殊标注 |
| 上一代旗舰（历史标杆） | ⚠️ 谨慎入库 | 必须标注「静态版本」+ 注明被替代时间 |
| 更早版本 | ❌ 不入库 | 除非有特殊场景价值（如某场景 Claude 3.5 仍优于 3.7） |
| 纯老版本（>1年） | ❌ 不入库 | P1 拦截 |

```
Step 1：搜「模型名 + official blog / release notes」
Step 2：在官方页面核验版本号和发布日期
Step 3：搜「模型名 + version history / changelog / timeline」
```

## 四、版本过时的判定标准

### P0 拦截条件（直接 FAIL）

- 收录了已发布替代版本的旧版本，却没有标注「静态版本」
- 版本号与实际发布日严重不符（如把 2023 年的模型标为 2024 年）

### P1 拦截条件

- 收录历史版本但未标注替代时间
- 搜索结果中官方已有新版，但调研报告未提及

### 软性处理（可降级入库）

- 某模型线青黄不接（>6个月无更新）→ 保留旧版，标注「截至 X 月最新版本」
- 旧版在特定场景仍优于新版 → 保留 + 场景说明

## 五、团队执行 Checklist（每次调研后必须自检）

- [ ] 我收录的是该模型线的哪个版本？
- [ ] 这个版本是官方当前最新版本吗？
- [ ] 官方是否有更新的版本已经发布？
- [ ] 我是否在报告中标注了版本发布日期？
- [ ] 我是否在报告中注明了是否有更新的替代版本？
- [ ] 我的信息源是否为官方来源（至少 T1 以上）？

## 六、WY 提到的具体问题回答

**Q：GPT-4o 已更新，3.6 你还会作为入库考虑么？**

Claude 3.6 的正确回答：
- Claude 3.6 是非真实存在的版本号。Anthropic 跳过了 3.6，直接发布 Claude 3.7。
- 如果调研报告中出现「Claude 3.6」，则该信息**本身不可靠**，需要返回 T0 信息源重新核实。
- 历史版本 Claude 3.5 Sonnet/Opus 可作为静态版本保留，Claude 3.6 若被提及则说明信息来源有问题。

**入库逻辑：**
- 当前最新（Claude 3.7）→ ✅ 入库，优先展示
- 历史旗舰（Claude 3.5 Opus）→ ⚠️ 可入库，标注「已被 3.7 替代」
- 虚构版本（Claude 3.6）→ ❌ 不入库，返回重查
