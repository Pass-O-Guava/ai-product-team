# 调研标准操作手册

> 所有专属调研智能体必须严格遵守本文档的所有标准。
> 本文档是 @researcher 反思整改的核心输出，也是 @coordinator 调度调研任务的执行依据。

---

## 一、信息源分级标准（强制）

| 优先级 | 来源 | 可用于 | 禁止 |
|--------|------|--------|------|
| T0（必须） | HuggingFace 官方页 + ModelScope 官方页 | 参数量/许可证/模态/链接 | — |
| T1（优先） | 官方 GitHub Release / 官方博客 | 发布日期/版本/SOTA声明 | — |
| T2（参考） | arXiv 论文 / 官方 Leaderboard | Benchmark 分数 | — |
| T3（补充） | 权威第三方媒体（有来源标注） | 场景/生态描述 | 无标注信息 |
| ❌ | 无来源 / 估算 / 推测 | 任何字段 | 任何关键字段 |

**关键规则**：
- 每个字段必须有对应优先级的来源支撑
- 低优先级来源必须注明，不得混用
- 无来源 = 不写

---

## 二、时间窗口检索规则（强制）

收到调研指令后，按以下顺序检索目标模型：

```
(a) 优先：最近1周内发布
    ↓ 无
(b) 扩展：最近1个月内发布
    ↓ 无  
(c) 再扩：最近1个季度内发布
    ↓ 无
(d) 极限：最近半年内发布
    ↓ 无
(e) 原则不收录（除非有重大技术突破意义）
```

**超过半年的模型**：不收录，已收录的标注"过时"标签

## 三、Model Card 必检清单（提交前强制填写）

调研员在提交 Model Card 前，必须逐项核对：

- [ ] **日期**：发布日期精确到日，来源链接是否附上？
- [ ] **T0 来源**：HuggingFace + ModelScope 链接是否实测可访问？
- [ ] **许可证**：是否明确标注商用合规（✅/⚠️/❌）？
- [ ] **Benchmark**：SOTA 声明是否附有量化分数 + benchmark 名称 + 来源链接？
- [ ] **自洽性**：可靠度评级与实际数据量是否匹配（无数据不得填A级）？
- [ ] **SOTA 声明**：每个 SOTA/最强 声明是否都有对应数据支撑？
- [ ] **缺失字段**：所有必填字段是否完整？

**全部通过才能提交**

---

## 四、Benchmark 数据要求

Benchmark 数据必须包含：
- 具体分数（如 MMMU: 73.2）
- Benchmark 名称（如 MMMU、MathVista、Video-MME）
- 评测条件（如 零样本 / 少样本 / 模型规模）
- 来源链接（官方 Leaderboard / 论文）

**禁止**：
- ❌ 只写"超越 GPT-4o"却不附分数
- ❌ 只写"领先"却不写具体对比对象
- ❌ 只写"很强"却无任何量化数据

---

## 五、可靠度评级标准

| 评级 | 含义 | 要求 |
|------|------|------|
| **A级** | 官方一手数据，权威可信 | 有 T0 或 T1 来源 + 具体 Benchmark 分数 |
| **B级** | 第三方验证，有参考价值 | 有 T2 来源 + Benchmark 数据 |
| **C级** | 估算/定性描述，无量化数据 | 仅 T3 来源或无来源的定性描述 |

**禁止**：无 Benchmark 分数却标注 A 级（自相矛盾）

## 六、Model Card 目录结构

```
models/
├── vision/          ← 视觉-语言模型（@vlm-researcher 负责）
├── audio/           ← 音频-语言模型（@alm-researcher 负责）
├── video/           ← 视频理解模型（@video-researcher 负责）
├── multimodal/      ← 多模态统一模型（@unified-researcher 负责）
└── text/            ← 纯语言模型（@text-researcher 负责）
```

每个模型一个 .md 文件，命名：`{model-id}.md`（如 `qwen3-vl.md`）

```

      ├── @vlm-researcher（视觉模型调研）
```

## 八、每日迭代机制

- **每日1-2次**调度调研智能体执行增量更新
- 新模型发布后，24小时内更新相关分类
- 旧模型版本更新时，同步更新对应 Model Card
- 每次更新记录到 update-log.md
