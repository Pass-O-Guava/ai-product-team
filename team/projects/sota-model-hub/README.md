# SOTA模型仓 · SOTA Model Hub

> 收录全球最新 SOTA 开源模型，提供横向对比，帮助用户快速解决业务需求。

---

## 🎯 产品定位

**SOTA模型仓** 是一个面向 AI 开发者和企业用户的开源大模型知识库。

- **是什么：** 一个结构化的 SOTA（State-of-the-Art）开源模型索引与对比平台
- **解决什么问题：** 帮助用户在海量开源模型中快速找到最优解，省去逐个调研的时间成本
- **数据来源：** HuggingFace + 魔搭社区（ModelScope），覆盖全球主流开源模型生态

## 🚀 如何使用

### 步骤 1：明确需求场景

| 场景 | 推荐路径 |
|------|---------|
| 想找一个能力最强的文本模型 | → `models/text/` → 对比各基准测试分数 |
| 需要视觉理解能力 | → `models/vision/` → 查看视觉模型列表 |
| 关注推理速度而非极致能力 | → `compare/methodology.md` → 查看推理速度维度 |
| 需要商用友好的模型 | → `compare/methodology.md` → 查看许可证维度 |

### 步骤 2：横向对比

进入对应的模型分类目录，查看该分类下的模型对比表。表格包含：

- **基准测试分数**（能力维度）
- **参数量 / 算力需求**（部署维度）
- **许可证类型**（合规维度）
- **最新版本 / 最后更新时间**（时效维度）

### 步骤 3：组合选型

如需多模型组合（如 RAG + 向量模型），参考各目录下的组合选型建议。

## 🔄 更新机制

如何保持 SOTA——我们这样保持知识库的时效性：

| 机制 | 说明 |
|------|------|
| **定期扫描** | 每周扫描 HuggingFace / ModelScope trending 榜单 |
| **版本追踪** | 对已收录模型持续追踪新版本发布 |
| **社区贡献** | 开放 PR 通道，欢迎社区提交新模型或更新数据 |
| **自动化提醒** | 基于模型发布动态触发更新任务 |
| **SOTA 刷新规则** | 当某模型在特定基准测试中刷新 SOTA 时，标记并更新对比表 |

## 🤝 贡献指南

### 方式一：提交新模型

1. 在对应分类目录下创建新的模型文档（参考 `_templates/model-card-template.md`）
2. 按模板填写模型信息：名称、来源、基准测试数据、许可证、适用场景等
3. 提交 Pull Request，注明数据来源

### 方式二：更新现有数据

- 基准测试分数更新 → 对应模型的文档中更新数据并注明来源
- 发现数据有误 → 提交 Issue 或直接 PR 修正

### 方式三：提出建议

- 缺少某个分类或评估维度？
- 希望增加某个特定模型的对比？

→ 提交 Issue 告诉我们

### 数据来源要求

- 必须标注原始来源（HuggingFace / ModelScope / 官方论文）
- 基准测试数据需注明评测环境和配置
- 避免引用未经验证的第三方评测结论
