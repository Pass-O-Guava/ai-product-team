# 调研团队 vs 质检团队 · 竞争与进化机制

> **建立时间**：2026-04-04  
> **建立者**：@pm  
> **基于**：WY 的团队设计理念

---

## 一、两个团队的定位与 KPI

| 团队 | KPI（核心指标） | 方向 |
|------|--------------|------|
| **@researcher 团队**（5个专属调研员） | 最终被质检通过、加入 SOTA 模型库的**模型数量** | 越多越好 |
| **@reviewer 团队**（知识质检智能体） | 拦截住的不符合要求的模型/信息**数量** | 越多越好 |

两个团队的 KPI 方向相反，但**终极目标完全一致**：让 SOTA 模型库越来越丰富、越来越专业、越来越权威、越来越有竞争力。

---

## 二、良性竞争机制

### 竞争维度

| 维度 | 调研团队 | 质检团队 |
|------|---------|---------|
| **核心目标** | 扩充 SOTA 模型库 | 发现并拦截错误 |
| **KPI** | 通过质检的模型数量 | 拦截的错误/不实信息数量 |
| **工作方式** | 主动出击，快速调研 | 严格审查，主动挑错 |
| **激励机制** | 通过越多，产出越多 | 拦截越多，质量越稳 |

### 进化逻辑

```
调研团队提交 → 质检团队拦截 → 调研团队整改 → 再次提交
     ↑                                              ↓
   通过 ✅                               持续迭代 🔄
     ↓                                              ↑
                 共同进化 → 质量提升
```

**良性竞争 = 互相推动，而不是互相拆台。**

## 三、质检门槛的灵活把控（重要原则）

> **原则：门槛要严格，但也要灵活。**

### 硬性门槛（绝对不能降）
- 信息准确性：数据必须与官方来源一致
- 日期精确性：发布时间必须精确到日
- SOTA 声明必须有量化证据
- 可靠度评级与数据完整性必须匹配

### 软性门槛（灵活处理）
- **时效性**：如果某一模态近半年内没有新模型、新突破，可以适当扩大检索时间窗口，但必须标注"静态版本"标签
- **Benchmark 缺失**：若模型确实无公开 benchmark 数据，可以降为 C 级可靠度，改为定性描述，但不得声称 SOTA
- **模型缺失**：若某模态确实无合适模型入库，不强制入库，以"暂无合适模型"标注

### 禁止"为了入库而入库"
- 不能因为要完成 KPI 就降低标准
- 宁缺毋滥，哪怕某个分类只有一个模型，只要质量够高

## 四、拦截指标（质检 KPI）

| 拦截类型 | 说明 |
|---------|------|
| **P0 拦截** | 日期造假、SOTA 无证据、自相矛盾——直接 FAIL |
| **P1 拦截** | 信息不完整、链接失效、许可证标注错误 |
| **P2 拦截** | 格式问题、表述不精准、过期信息 |

**质检 KPI**：累计拦截 P0/P1/P2 数量，每月统计一次，向 @pm 汇报。

## 五、与 HuggingFace / ModelScope 的差异化竞争

> **核心问题：我们跟 HuggingFace 和魔搭社区的信息差异在哪里？我们独有的竞争力到底是什么？**

### 我们提供，而它们不提供的

| 维度 | HuggingFace | ModelScope | **我们的竞争力** |
|------|------------|-----------|----------------|
| **SOTA 标注** | ❌ 无官方标注 | ❌ 无 | ✅ 精选 SOTA，每日更新，附量化证据 |
| **Benchmark 对比** | ❌ 无横向对比 | ❌ 无 | ✅ 多模型横向对比表，按维度选型 |
| **中文视角** | 英文为主 | 中文为主 | ✅ 中文 + 全球视野 + 中国客户视角 |
| **商用合规指南** | ❌ 无统一整理 | 部分 | ✅ 许可证商用合规标签，商用决策辅助 |
| **组合推荐** | ❌ 无 | ❌ 无 | ✅ 按场景推荐模型组合（RAG+向量等） |
| **质检报告** | ❌ 无 | ❌ 无 | ✅ 每模型经 @reviewer 严格审核，有据可查 |
| **持续更新** | 被动更新 | 被动更新 | ✅ 主动调研，每日迭代，半月沉淀 |
| **行业洞察** | ❌ 无 | 较弱 | ✅ 行业技术趋势分析，选型建议 |

### 我们的变现逻辑

```
高质量 SOTA 知识库
    ↓
客户愿意为准确、权威的选型参考付费
    ↓
可变现形式：
- 会员订阅（定期收到更新报告）
- 付费选型咨询（基于知识库的专家服务）
- 企业定制调研（针对特定行业的模型选型报告）
- API 集成（模型能力 API 查询服务）
```

## 六、两个团队的共同目标

> **让 SOTA 模型库成为一个高质量、准确、具有行业技术洞察力的模型库。**
> **这个库拿出去，就是有客户愿意为它买单的。**

两个团队的一切分歧，都必须在"这个目标"面前让路。

## 七、汇报与复盘机制

- **每日**：@coordinator 汇总调研进度，向 @pm 汇报
- **每周**：@reviewer 出具质检周报（拦截数量 / 通过数量 / 典型问题）
- **每月**：团队复盘会议，两个团队共同参与
