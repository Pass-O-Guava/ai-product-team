# 模型对比维度与 SOTA 判断标准

> 文档版本：v0.1 | 更新日期：2026-04-04

---

## 1. 评估维度总览

我们在对比模型时，从以下六个核心维度进行评估：

| 维度 | 说明 | 权重（参考） |
|------|------|------------|
| **能力表现** | 基准测试分数 | ⭐⭐⭐⭐⭐（最高） |
| **推理速度** | Token/s、延迟 | ⭐⭐⭐⭐ |
| **许可证** | 开源协议与商用限制 | ⭐⭐⭐⭐ |
| **算力需求** | 显存 / 内存需求 | ⭐⭐⭐ |
| **易用性** | 部署难度、社区生态 | ⭐⭐⭐ |
| **多模态支持** | 是否支持多模态 | ⭐⭐⭐ |

---

## 2. 各维度详细说明

### 2.1 能力表现（Capability）

通过标准基准测试（Benchmark）衡量模型的认知与推理能力。

| 基准测试 | 适用模态 | 考察能力 |
|---------|---------|---------|
| MMLU | 文字 | 多学科知识理解 |
| HumanEval / MBPP | 文字 | 代码生成 |
| GSM8K | 文字 | 数学推理 |
| BBH (Big Bench Hard) | 文字 | 复杂推理 |
| C-Eval / CMMLU | 文字 | 中文能力 |
|VQAv2 / TextVQA | 视觉 | 视觉问答 |
| COCO (mAP) | 视觉 | 目标检测/分割 |
| DocVQA | 视觉 | 文档理解 |
| LibriSpeech (WER) | 音频 | 语音识别 |
| SLUE | 音频 | 语音理解 |

> **记录规则**：记录官方评测结果，注明评测配置（few-shot / CoT 等），并标注数据时间。

---

### 2.2 推理速度（Inference Speed）

| 指标 | 说明 | 常用工具 |
|------|------|---------|
| **Throughput** | Tokens/s（端到端） | vLLM / LM Evaluation Harness |
| **首 Token 延迟 (TTFT)** | Time to First Token | 实际部署测试 |
| **单次推理延迟** | 单次请求总延迟 | 同上 |
| **吞吐量（批处理）** | Batch 模式下的并发吞吐 | 同上 |

> **记录规则**：记录硬件配置（如 A100 / H100 / RTX 4090），标注输入/输出长度，以便跨模型对比。

---

### 2.3 许可证（License）

| 许可证类型 | 典型代表 | 商用许可 | 商用限制 |
|-----------|---------|---------|---------|
| **MIT** | LLaMA 2（部分） | ✅ 自由商用 | 无 |
| **Apache 2.0** | Mistral 系列 | ✅ 自由商用 | 无 |
| **Llama 2 License** | Llama 2 | ⚠️ 有条件商用 | 月活 > 700M 需申请 |
| **CC BY-NC 4.0** | 部分学术模型 | ❌ 不可商用 | 仅学术用途 |
| **CC BY-SA 4.0** | 部分开源模型 | ⚠️ 商用需相同协议 | 衍生品需开源 |
| **Proprietary** | GPT-4 等 | ❌ 不可用 | 仅 API 调用 |
| **ODC-By** | 开放数据许可证 | ✅ 需署名 | 需注明来源 |

> **记录规则**：以 HuggingFace / ModelScope 模型卡页面标注的许可证为准，注明许可证全文链接。

---

### 2.4 算力需求（Hardware Requirements）

| 指标 | 说明 |
|------|------|
| **参数量** | 模型总参数量（7B / 13B / 70B 等） |
| **FP16 显存需求** | 半精度推理所需显存 |
| **INT8 显存需求** | 量化推理所需显存 |
| **INT4 显存需求** | 更激进的量化 |
| **最低内存** | CPU 推理或 offload 模式 |
| **推荐 GPU** | 最小可运行 GPU 配置 |

> **记录规则**：记录官方文档中的最低/推荐配置，并标注测试环境。

---

### 2.5 易用性（Usability）

| 指标 | 说明 |
|------|------|
| **HuggingFace 下载量** | 反映社区热度 |
| **GitHub Stars** | 反映开发者关注度 |
| **官方文档完善度** | 有无完整 README / API 文档 |
| **量化支持** | 是否支持 GGUF / GPTQ / AWQ |
| **推理框架兼容** | vLLM / llama.cpp / TGI 等 |

---

### 2.6 多模态支持（Multimodal）

| 能力 | 说明 |
|------|------|
| **图像理解** | 是否支持图像输入 |
| **语音输入/输出** | ASR / TTS 能力 |
| **视频理解** | 是否支持视频帧输入 |
| **工具调用（Function Calling）** | 支持的外部工具接口 |
| **Agent 能力** | 是否有内置 Agent 框架 |

## 3. SOTA 判断标准

### 3.1 什么是 SOTA

SOTA = **State-of-the-Art**，指在某一特定维度上达到当前最优性能的模型。

> 注意：**SOTA 是维度性的，不是全局性的**。没有一个模型在所有维度上都是 SOTA。

### 3.2 SOTA 判断规则

| 条件 | 说明 |
|------|------|
| **规则 1：基准测试最高分** | 在某基准测试中排名#1，且分数显著领先（> 3%） |
| **规则 2：发布时效性** | 新模型发布后刷新某基准记录，原 SOTA 自动降级为"前 SOTA" |
| **规则 3：综合维度加权** | 综合评分最高（按权重加权），在能力维度权重最高 |
| **规则 4：独立验证** | 分数须经独立第三方评测验证（非仅官方自评） |

### 3.3 SOTA 标记格式

在模型卡片中，SOTA 模型按如下方式标记：

```
🏆 SOTA in [维度] — [基准测试名称] — [分数] — [日期]
```

示例：
```
🏆 SOTA in 中文推理 — C-Eval — 92.3 — 2026-03
```

## 4. 数据来源说明

| 数据类型 | 来源 | 优先级 |
|---------|------|--------|
| 模型权重地址 | HuggingFace / ModelScope 官方 | 必须 |
| 基准测试分数 | 官方论文 / 官方 GitHub | 优先 |
| 第三方评测 | LMSYS Chatbot Arena / Open LLM Leaderboard | 参考 |
| 许可证信息 | 模型卡（Model Card）License 字段 | 必须 |
| 硬件需求 | 官方文档 / GitHub README | 优先 |

### 来源可靠性评级

| 评级 | 说明 |
|------|------|
| 🟢 **官方自测** | 来自模型官方发布，标注完整测试配置 |
| 🟡 **第三方独立** | 来自权威第三方评测平台 |
| 🔴 **估算/参考** | 社区估算，无完整测试数据，标注"待验证" |
