## 三、调研标准（强制，所有调研员遵守）

### 3.1 信息源优先级（强制分级）

| 优先级 | 来源 | 用途 |
|--------|------|------|
| T0（必须） | HuggingFace 官方页 + ModelScope 官方页 | 参数量/许可证/模态/链接 |
| T1（优先） | 官方 GitHub / 官方博客 | 发布日期/版本/SOTA声明 |
| T2（参考） | arXiv 论文 / 官方 Leaderboard | Benchmark 数据 |
| T3（补充） | 权威媒体（有来源标注） | 场景/生态描述 |
| 禁止 | 无来源标注的信息 | 任何关键字段 |

### 3.2 时间窗口检索规则

收到指令后按以下顺序扩展：
- (a) 最近1周 → (b) 最近1个月 → (c) 最近1季度 → (d) 最近半年
- **超过半年的模型，原则上不收录**（有重大技术突破可例外）

### 3.3 每模型输出要求

每模型必须输出一份 Markdown 分析报告，包含：
1. 基本信息（名称/发布方/参数量/模态/发布日期精确到日）
2. 许可证与商用合规（✅可商用 / ⚠️需申请 / ❌不可商用）
3. HuggingFace + ModelScope 链接（必须实测可访问）
4. Benchmark 量化数据（附分数/benchmark名称/来源链接）
5. SOTA 声明（必须有量化证据，无证据不得声称 SOTA）
6. 技术亮点（有据可查的能力描述）
7. 入选理由（为什么选这个模型）
8. 信息源列表

## 五、调研员调研纪律（强制，七条）

1. **数据优先，结论滞后**：无数据的结论不上线
2. **日期精确到日，来源必须可查**：模糊日期=P0错误
3. **可靠度评级与信息完整性必须匹配**：无分数不得填A级
4. **每个字段写完必须自检**：五项检查表未通过不上线
5. **信息源有等级，优先顺序不能乱**：禁止使用未标注来源
6. **SOTA声明双人背书**：无背书不上线
7. **链接必须实测**：禁止"应该能访问"

## 六、质检团队 KPI 与竞争机制

### 两个团队 KPI（对立统一）

| 团队 | KPI | 方向 |
|------|------|------|
| 调研团队（5个专属调研员） | 通过质检的模型数量 | 越多越好 |
| 质检团队（@reviewer） | 拦截的错误/不实信息数量 | 越多越好 |

### 质检拦截等级

| 等级 | 说明 |
|------|------|
| P0 | 日期造假、SOTA 无证据、自相矛盾 → 直接 FAIL |
| P1 | 信息不完整、链接失效、许可证标注错误 |
| P2 | 格式问题、表述不精准、过期信息 |

### 准入灵活把控原则

**硬性门槛（绝对不降）：** 信息准确性、日期精确性、SOTA 证据链

**软性门槛（灵活处理）：**
- 某模态近半年无新模型 → 扩大窗口 + 标注"静态版本"标签
- 模型无公开 benchmark → 降为 C 级可靠度，不得声称 SOTA
- 确实无合适模型 → 以"暂无合适模型"标注，不强制入库

**禁止**：为完成 KPI 而降低标准，宁缺毋滥

## 十一、分类逻辑规范（2026-04-05 强制执行）

**核心原则：开源与闭源绝不混列，同一模型不得同时出现在两类中。**

| 分类维度 | 说明 |
|---------|------|
| 许可证类型 | 以模型实际许可证为准，HuggingFace 页面核实 |
| 闭源定义 | 专有模型（API 调用，不可下载权重） |
| 开源定义 | 有公开权重，可本地部署（MIT/Apache 2.0/BSD/DeepSeek License/Tongyi Qianwen 等） |

**当前 10 大分类**：闭源文本与推理模型、开源文本大模型、闭源 VLM、开源 VLM、音频-语言模型、视频理解模型、图像生成模型、编程/推理专项、多模态统一模型、Embedding & Reranker
