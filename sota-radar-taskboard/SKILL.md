# 任务时间追踪 Skill

> 职责：为每个任务建立时间记录，精确到分钟。
> 触发时机：每次派发子智能体任务时，每次任务完成时。
> 记录位置：`/workspace/sota-radar-taskboard/tasks.json`

---

## 任务记录规范

每个任务必须记录：

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 任务唯一标识 | `v6.2` |
| title | 任务描述 | `V6.0: OpenClaw Agent接入` |
| owner | 负责人 | `子智能体-A` |
| status | 状态 | `待开始/进行中/已完成/已阻塞` |
| created_at | 创建时间（UTC） | `2026-04-06T01:36:00Z` |
| started_at | 开始时间（UTC） | `2026-04-06T01:36:00Z` |
| finished_at | 结束时间（UTC） | `2026-04-06T02:15:00Z` |
| duration_min | 实际耗时（分钟） | `39` |
| version | 版本 | `V6.0` |
| notes | 备注 | `发现卡点：需要API token` |
| blockers | 阻塞原因数组 | `["无服务器权限"]` |

---

## 派发任务标准流程

```
1. 创建任务记录（status: 进行中，started_at: NOW）
2. 派发子智能体
3. 子智能体完成后：更新 finished_at + duration_min + status: 已完成
4. 如有阻塞：status: 已阻塞，附上blockers
5. 立即更新前端任务看板（重部署或动态加载）
```

---

## 时间记录格式（UTC）

```bash
# 获取当前UTC时间
date -u '+%Y-%m-%dT%H:%M:%SZ'

# 例如：2026-04-06T01:36:00Z
```

---

## 任务状态说明

| 状态 | 含义 | 行动 |
|------|------|------|
| 待开始 | 已规划，未开始 | 派发 |
| 进行中 | 正在执行 | 每15分钟检查一次 |
| 已完成 | 正常结束 | 更新记录，通知WY |
| 已阻塞 | 遇到障碍 | 立即汇报WY，不等待 |

---

## 汇报规则

**必须立即汇报（不等待）：**
- 任务状态变为"已完成"时
- 任务状态变为"已阻塞"时
- 发现比预期多花50%以上时间时

**汇报格式：**
```
📋 任务 [v6.1] 完成
- 耗时：39分钟（原计划30分钟，超时30%）
- 完成内容：...
- 下一步：自动进入V7.0
```
